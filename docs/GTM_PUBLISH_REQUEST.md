# GTM タグ公開依頼（クライアント様送付用）

最終更新: 2026-05-13

---

## 背景

2026-05-12 のレビューで、Looker Studio の以下指標が常に 0 または null になっている件を調査いたしました。

| 症状 | 関連指標 |
|------|---------|
| STEP4(入力開始)=0% / STEP5(送信完了)=4.41% の論理矛盾 | `form_start` イベント |
| ページ別の `scroll_90pct_rate` が全 0 | `scroll_depth` イベント |
| ページ別の `cta_click_rate` が全 0 | `cta_click` イベント |

BQ の実測（過去30日）でも `scroll_depth` / `cta_click` / `form_start` の発火件数は 0 件でした。

調査の結果、**GTM コンテナにタグは設置されているが「公開」ボタンが押されておらず、プレビュー環境のみで稼働している状態**と推測しております。

> BQ 側の集計ロジックは並行して修正済み（GTM 送信するイベント名 `scroll_depth` / `cta_click` と整合）。`~/projects/ark-analytics/` master ブランチ参照。

---

## ご依頼内容（5分作業・所要時間: 1〜2 営業日待機含む）

### STEP 1: GTM コンテナにログイン

1. https://tagmanager.google.com/ にアクセス
2. コンテナ ID: **GTM-XXXXXXX**（KNOWLEDGE.md 記載）を選択

### STEP 2: 既存タグ ① ② ③ の存在確認

タグ一覧で下記 3 つのタグが **登録済み** であることを確認してください。タグコードは既に `docs/GTM_TAGS.md` で共有済みのものです。

| # | タグ名（任意） | 種別 | トリガー |
|---|--------------|------|---------|
| ① | スクロール深度 | カスタムHTML | All Pages |
| ② | CTAクリック | カスタムHTML | All Pages |
| ③ | フォーム操作 | カスタムHTML | `/contact/` 含むページ |

タグが見つからない場合は弊社まで一度ご連絡ください（タグコードを再共有いたします）。

### STEP 3: プレビューで動作確認

1. 右上「プレビュー」をクリック
2. 確認用 URL を貼って example.invalid を開く
3. 「フォーム入力欄をクリック」「ページを90%までスクロール」「お問い合わせCTAをクリック」を試す
4. プレビュー画面に `form_start` / `scroll_depth` / `cta_click` の3イベントが表示されることを確認

### STEP 4: 公開

1. 右上「送信」をクリック
2. バージョン名: `2026-05-13 タグ①②③公開` 等
3. 「公開」ボタンを押す

### STEP 5: 24-48時間後にご連絡

GA4 → BigQuery への自動エクスポートは翌日反映のため、24-48 時間後にご一報いただければ弊社で BQ 側のイベント発火状況を確認し、Looker Studio の数値が正常に流れることをご報告いたします。

## ご担当者様への確認方法（オフラインで自己検証する場合）

GA4 管理画面の「イベント」一覧で、公開から数時間後に以下が表示されれば成功です：

- `scroll_depth`（scroll_pct パラメータ付き）
- `cta_click`（cta_location / cta_type / cta_purpose パラメータ付き）
- `form_start`
- `contact_finish`（既存）

## 補足: なぜ SQL 側だけでは解決できないか

GTM タグが未公開の場合、ブラウザ → GA4 → BigQuery の経路でイベント自体が送信されません。BQ 側でいくら集計ロジックを修正しても、元データが 0 件のままなので可視化できる数値も 0 のままになります。

GTM タグの公開と BQ 側 SQL の整合性修正の **両方が揃って初めて**ファネル分析・スクロール率・CTA率の計測が正常化します。
